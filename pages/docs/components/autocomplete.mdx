# Autocomplete

Autocomplete component is using `Combobox` component from
[headless ui](https://headlessui.com/react/combobox) library.

## Instalation

import { Tab, Tabs } from 'nextra-theme-docs'
import { SCRIPTS_TABS } from '../../../constants'

<Tabs items={SCRIPTS_TABS}>
  <Tab>
    ```bash copy yarn add @creation-ui/core @creation-ui/autocomplete ```
  </Tab>
  <Tab>```bash copy npm i @creation-ui/core @creation-ui/autocomplete ```</Tab>
</Tabs>

## Usage

## Basic usage example

```tsx copy
import { Autocomplete } from '@creation-ui/autocomplete'

const [Autocompleteion, setAutocompleteion] = useState<Option | never>()

return (
  <Autocomplete
    size='sm'
    options={options}
    label={'Autocomplete'}
    value={Autocompleteion}
    onChange={setAutocompleteion}
  />
)
```

## Autocompleteing multiple options

In this example, we are using the `multiple` prop to allow Autocompleteing multiple options.

<br />
<AutocompleteMultipleExample
  clearable
  clearableButtonText='Clear Autocompleteion'
/>
<br />
Because of the usage of the Listbox component under the hood, the `value` and `onChange`
props change the signature:{' '}

| Property | `!multiple` | `multiple`    |
| -------- | ----------- | ------------- |
| value    | `T`         | `T[]`         |
| onChange | `T => void` | `T[] => void` |

For details see [properties table](#onChange)

## Error state

```js copy

const [Autocompleteion, setAutocompleteion] = useState<Option | never>()

  return (
    <Autocomplete
      clearable
      clearableButtonText="Clear Autocompleteion"
      options={options}
      label={'With error'}
      value={Autocompleteion}
      onChange={setAutocompleteion}
      error={'You must Autocomplete an option'}
    />
  )

```
